Git Intro

Why Git?

  • We need a nice way to keep track of changes to code
  • We want a way to revert back to previous versions of code

Local Workflow

  • Three areas
    • Working
    • Staging
    • Clean

Essential Commands

  • git init
  • git status
  • git add <NAME_OF_FILE>
  • git commit -m
  • git log

Creating Remote Repositories on GitHub

  • Navigate to https://www.github.com/new
  • Follow the second block of instructions
  • Instead of using a remote called origin, call the remote your first name.
  • Confirm you have created a remote correctly using git remote -v

Whatâs a remote?

  • Itâs a nickname for a URL where your repository lives!
  • Instead of typing/remembering the URL every time, we give it a nickname
  • git remote add NAME_OF_REMOTE URL_FOR_REPOSITORY

Pushing Your Code

The command we use is git push NAME_OF_REMOTE NAME_OF_BRANCH

Working with Others

Pull vs Clone

  • When working with others you have two options for workflow
    • Cloning and pushing (requires collaborator access)
    • Forking, Cloning, Submitting a Pull Request (PR)

Weâre going to start with cloning and pushing

  • When working with others, make sure you give them collaborator access
  • Since they will be able to write to the repository, make sure you always communicate

How to work with your pair

  • Once you have finished an assignment and you have given your pair collaborator access. Your collaborator must do the following:
    • Clone your repository
    • Make a new repository on their github account
    • Instead of adding a remote called origin, add a remote by their first name
    • In the terminal, git push THEIR_NAME master
