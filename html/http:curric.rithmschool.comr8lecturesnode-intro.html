Intro to Node.js

What is Node.js?

  • An JavaScript environment that runs server-side
    • It uses the Chrome v8 engine, but doesnât require/use Chrome
  • Can be used to build any kind of server-side JS
    • Including building web applications!

Why Node.js?

  • You can do all your programming in JavaScript
  • Widely used
  • There is an extensive set of add-on libraries via npm
    • Many can also be used in client-side JS

Starting Node

$ node myScript.js
Hi there!

or, for an interactive REPL:

$ node
>

(Control-D to exit)

NPM

What is NPM?

  • Massive repository of add-on libraries
  • Command line tool, npm, comes with Node
  • Keeps track of dependencies so others can quickly get your deps
  • Register with npm to publish open-source packages

Starting a Project with NPM

$ cd my-project

$ npm init
  • Creates package.json with metadata & dependencies
  • In a hurry? Want the defaults? npm init -y

Example package.json

{
  "name": "node-files",
  "version": "1.0.0",
  "description": "Exercise to create file listing functions.",
  "main": "",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "Whiskey the Dog",
  "license": "ISC",
  "dependencies": {
    "axios": "^0.18.0"
  }
}
  • Store your package.json in Git

Installing Packages

$ npm install axios
...
  • Adds to package.json
  • Installs all dependencies to node_modules
    • Put node_modules in .gitignore

To install everything already defined in package.json:

$ npm install
...

package-lock.json

  • package.json contains metadata & dependency requirements
    • If you installed as npm install 'axios@>=0.16',
      this would have ârequires version at least as new as 0.16â
  • package-lock.json contains exact versions obtained
    • Shows â0.18.0 installedâ
  • If you want setup just like yours, keep package-lock.json in Git

Node Callbacks

Many Node library functions use callbacks

For example, to read a file:

node-intro-demo/read.js

  if (err) {
    console.error(err);
  }
  console.log(`file contents: ${data}`);
});

// file won't have been read yet at this point

Handling Errors

In web JS, there are different things to do with errors:

  • Show some âan error happenedâ message in the DOM
  • Pop up an alert box
  • Log to the console

In Node, you will often do one of these:

  • Log to the console
  • Exit the program with process.exit(1)
    • You need to import the process library
