demo/badhash.py
def awful_hash(phrase):
    """Truly terrible hash: simply shifts each letter (a->b, etc).

        >>> awful_hash('yay')
        'zbz'
    """

    return ''.join(next_char(c) for c in phrase)

But is that really one-way?

One-Way Encryption

demo/badhash.py
def slightly_better_hash(phrase):
    """Better hash: returns every other letter, shifted, max 4.

        >>> slightly_better_hash('penguin1')
        'qovo'

    Since this is "lossy", multiple inputs can return same output:

        >>> slightly_better_hash('penguin1~pretzel7')
        'qovo'

        >>> slightly_better_hash('p?nguinZ')
        'qovo'
    """

    return ''.join(next_char(c) for c in phrase[0:8:2])
  • Now is one-way (non-reversible)
  • Same input always equal same output

Python has this kind of hash built-in:

>>> hash('penguin1')
6678229702981429425

(Pythonâs built in hash seeds itself randomly on startup, so the same input only returns the same output for any individual Python process. As such, itâs not suitable for storing in a database, even if it were designed to be cryptographically secure.)

Work Factor

  • Bcrypt algorithm is designed to be slow
    • But computers get faster all the time!
  • So, you can specify how many rounds of encryption it should use
    • And, over time, increase this âwork factorâ of work
>>> salt = bcrypt.gensalt(rounds=14)   # default (as of 4/2018) is 12

This is encoded in the result:

Class Methods

Registering

demo/goodpasswd/app.py
class User(db.Model): # ...

    @classmethod
    def register(cls, username, password):
        """Register a user, hashing their password."""

        hashed = bcrypt.generate_password_hash(password)
        hashed_utf8 = hashed.decode("utf8")
        return cls(username=username, password=hashed_utf8)
demo/goodpasswd/app.py
class User(db.Model): # ...

    @classmethod
    def authenticate(cls, username, password):
        """Validate that user exists & password is correct.

        Return user if valid; else return False.
        """

        user = User.query.filter_by(username=username.lower()).one()
        if user:
            if bcrypt.check_password_hash(user.password, password):
                return user

        return False

Using Class Methods

>>> db.session.add(nora)
>>> db.session.commit()

Staying Logged In

How do we remember a logged in user?

A magic dictionary specific to this user - the session!

demo/goodpasswd/app.py
@app.route('/color', methods=['GET', 'POST'])
def color():
    """Remember user's favorite color."""

    if request.method == 'POST':
        color = request.form.get('color')
        session['color'] = color
        return redirect("/color")

    else:
        color = session.get("color", "(dunno)")
        return render_template("color.html", color=color)

Session Review

  • Can store âfundamentalâ Python objects in session
    • Strings, integers, dictionaries, etc
  • Donât store âtransientâ things there
    • Database objects, etc
  • What can we store about a user to show theyâre logged in?
    • The user ID (or username, if itâs unique)

Logging in a user using the session

@app.route('/login', methods=['GET', 'POST'])
def login():
    """Produce login form or handle login."""

    if request.method == "POST":
        username = request.form.get("username")
        password = request.form.get("password")

        user = User.authenticate(username, password)  # <User> or False
        if user:
            session["user_id"] = user.id # this is how we log a user in!
            return redirect("/secret")

    return render_template("login.html")
